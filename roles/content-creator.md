# 内容创作者的 AI 工具箱

> 用 OpenClaw 搭建你的 AI 编辑部：自动追热点、辅助写稿、一键多平台分发。

## 你的痛点

- 每天花 1-2 小时刷各种信息源找选题，效率低
- 写完文章还要手动复制到公众号、小红书、即刻，格式每次都要调
- 想追热点但总是慢半拍，等你写完别人都发过了
- 素材散落在各处，用的时候找不到
- 一个人干三个人的活：采集、写作、发布

## 推荐方案

### 第一步：让 AI 帮你盯热点

**解决什么**：不用自己刷信息源，AI 自动监控并汇总。

**用什么**：

| Skill | 作用 |
|-------|------|
| `blogwatcher` | 监控 RSS/博客更新，自动发现新内容 |
| `searxng` | 隐私搜索，多引擎聚合，深挖某个话题 |

**怎么装**：
```bash
# blogwatcher 是 OpenClaw 内置，无需安装
# searxng 需要安装
openclaw skills install searxng
```

**效果**：设定好监控源后，每天自动抓取更新。你只需要问 agent "今天有什么值得写的"，它会给你筛选后的选题清单。

---

### 第二步：让 AI 帮你整理素材

**解决什么**：看到好文章/好内容，快速提取核心信息，不用自己一篇篇读。

**用什么**：

| Skill | 作用 |
|-------|------|
| `fetch_url_text` | 从网页提取正文，去掉广告和导航 |
| `summarize` | 长文摘要，快速了解核心内容 |
| `media_transcribe` | 把播客/视频转成文字，变成可用素材 |
| `youtube_transcribe` | YouTube 视频直接转文字 |

**怎么装**：
```bash
# summarize 是 OpenClaw 内置
# 其他可能需要手动配置，参考各 skill 评测
```

**效果**：给 agent 一个链接，它帮你提取正文 + 生成摘要。给一个视频链接，它帮你转成文字稿。你的素材库自动填充。

---

### 第三步：让 AI 辅助写稿

**解决什么**：有了素材，快速生成初稿，你来把关和润色。

**用什么**：

这一步主要靠 OpenClaw agent 本身的写作能力，配合一些辅助 skill：

| Skill | 作用 |
|-------|------|
| `boost-prompt` | 优化你给 AI 的写作指令，提升输出质量 |
| `nano-pdf` | 把内容导出为 PDF |

**效果**：告诉 agent 选题方向 + 素材，它输出初稿。你修改润色后定稿。从素材到初稿，10 分钟。

---

### 第四步：多平台分发

**解决什么**：写完一次，自动适配不同平台格式并发布。

**用什么**：

| Skill | 作用 |
|-------|------|
| `slack` | 发到 Slack 频道 |
| `discord` | 发到 Discord 频道 |

> ⚠️ 当前中文平台（公众号、小红书、即刻）的发布 Skill 还在开发中，这也是 Skill Hub 要填补的空白。

**现阶段替代方案**：
- 公众号：agent 帮你生成 Markdown 格式稿件 → 你手动粘贴到公众号编辑器
- 小红书：agent 帮你把长文改写成小红书风格短文 + 生成配图文案

---

### 第五步（可选）：把文章变成音频

**解决什么**：一份内容，多种形态。文字变播客/音频。

**用什么**：

| Skill | 作用 |
|-------|------|
| `azure-tts` | 文字转语音，多种音色可选 |
| `speak-text` | 快速文字转语音 |
| `article-to-audio` | 长文自动分段转音频 |

**效果**：文章写完后，一句话让 agent 生成音频版本，可以发播客或做视频旁白。

## 配置建议

1. **搜索默认用 SearXNG**：在 MEMORY.md 中写 `搜索优先使用 searxng`
2. **设定监控源**：用 blogwatcher 监控你关注的 RSS 源
3. **建立素材目录**：在 workspace 下建 `materials/` 目录，agent 自动把素材存这里

## 注意事项

- blogwatcher 需要配置 RSS 源地址，国内很多网站没有 RSS，需要借助 RSSHub
- 音视频转录需要本地 whisper 模型，首次运行会下载模型文件（约 1-3GB）
- AI 写的初稿一定要人工审核，不要直接发布
