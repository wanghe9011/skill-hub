# 评测标准

每个 Skill 从以下 6 个维度评分，每项 1-5 分。

## 🔧 可用性（Usability）

能否顺利安装和运行。

| 分数 | 说明 |
|------|------|
| 5 | 零依赖或依赖自动安装，一键可用 |
| 4 | 依赖明确，按文档操作即可成功 |
| 3 | 需要手动处理部分依赖或配置 |
| 2 | 安装过程有坑，需要踩坑才能跑通 |
| 1 | 无法正常安装或运行 |

## 🎯 有效性（Effectiveness）

实际触发后能否完成描述的功能。

| 分数 | 说明 |
|------|------|
| 5 | 完全符合描述，输出质量高 |
| 4 | 基本完成功能，偶尔有小问题 |
| 3 | 核心功能可用，但有明显局限 |
| 2 | 功能不完整，只能部分使用 |
| 1 | 基本不可用或效果很差 |

## 🛡️ 安全性（Security）

是否有可疑指令或安全风险。

| 分数 | 说明 |
|------|------|
| 5 | 纯指令型，无 exec/网络调用，完全安全 |
| 4 | 有 exec 但指令明确、范围可控 |
| 3 | 有外部 API 调用但行为透明 |
| 2 | 存在潜在风险，需谨慎使用 |
| 1 | 发现可疑指令（提权、数据外传等） |

## 📖 文档质量（Documentation）

SKILL.md 写得是否清晰、规范。

| 分数 | 说明 |
|------|------|
| 5 | 结构清晰，示例丰富，新手友好 |
| 4 | 说明完整，能理解怎么用 |
| 3 | 基本能用，但缺少示例或细节 |
| 2 | 信息不全，需要自己摸索 |
| 1 | 几乎没有文档或文档有误 |

## 🔄 维护状态（Maintenance）

是否持续更新和维护。

| 分数 | 说明 |
|------|------|
| 5 | 官方内置 / 活跃维护（近 1 个月有更新） |
| 4 | 定期维护（近 3 个月有更新） |
| 3 | 低频维护（近 6 个月有更新） |
| 2 | 基本停滞（超过 6 个月无更新） |
| 1 | 已弃用或作者消失 |

## 🌐 中文适配（Chinese Support）

是否支持中文使用场景。

| 分数 | 说明 |
|------|------|
| 5 | 原生中文 Skill，专为中文场景设计 |
| 4 | 核心功能支持中文输入输出 |
| 3 | 可用于中文场景但无专门适配 |
| 2 | 中文支持有限，部分功能不兼容 |
| 1 | 不支持中文 |

## 综合评级

| 评级 | 平均分 | 含义 |
|------|--------|------|
| ⭐⭐⭐⭐⭐ | 4.5+ | 强烈推荐 |
| ⭐⭐⭐⭐ | 3.5-4.4 | 推荐使用 |
| ⭐⭐⭐ | 2.5-3.4 | 可以一试 |
| ⭐⭐ | 1.5-2.4 | 不推荐 |
| ⭐ | < 1.5 | 避坑 |

## 特殊标签

- ✅ **已验证** — 经过完整实测
- ⚠️ **需谨慎** — 存在已知问题
- 🚫 **不推荐** — 有安全风险或完全不可用
- 🇨🇳 **中文原生** — 专为中文场景设计
